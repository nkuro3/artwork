# 02. プロトタイプ 共通スタイル（デザイン基盤）仕様書

## 1. 概要

プロトタイプ全体で共通して使う**スタイルの基盤**を定める。視覚デザイン（ブランド配色・イラスト・凝った装飾）はこの段では作らず、**タイポグラフィ・余白・機能色などの最小トークン**と、**全画面共通のレイアウト/状態/遷移/レスポンシブ/アクセシビリティの方針**を定義する。各画面の個別設計は本仕様の範囲外で、`03`（画面設計）で定める。

### 成功条件
1. デザイン基盤トークン（フォント/サイズ/余白/機能色/レイアウト）が定義され、全画面に一貫適用できる。
2. 共通レイアウト・状態・遷移・レスポンシブ・a11y の方針が定義され、各画面が参照できる。
3. 装飾的ビジュアルデザインなしでも、機能色とレイアウトだけで操作・閲覧できる。

### 前提
| 前提 | 内容 |
|---|---|
| 機能基盤 | 認証・作品CRUD・画像・ポートフォリオ・検索の API/データは 01-prototype で実装済み。本仕様はスタイル基盤のみを扱う |
| 配色・ブランド | ブランド配色・装飾は作らない。ただし機能色（文字/罫線/エラー/リンク/フォーカス）の最小セットは定義して操作性を担保 |

## 2. スコープ

### In scope
- デザイン基盤トークン（タイポグラフィ・余白・レイアウト・機能色・フォーカス/操作性・角丸・コンポーネント寸法/状態・グリッド列数）
- 全画面共通のレイアウト/ナビゲーション、状態・遷移・レスポンシブ・a11y の共通方針

### Out of scope
- 各画面の個別設計（要素・状態・遷移・文言）→ `03`（画面設計）
- ブランド配色パレット・アイコンセット・影/エレベーション・凝ったモーション
- 新規ドメイン機能（課金・ソーシャル等）
- 画像編集・トリミング等の高度な操作 UI
- ダークモード
- 実装手段（Tailwind/CSS 設計）の確定 — トークンの「値」のみ定義し、適用手段は実装時

## 3. 用語

| 用語 | 意味 |
|---|---|
| トークン | フォントサイズ・余白・色などの再利用可能な定義値 |
| 空状態 | データが0件のときの表示 |
| コンテナ | 中央寄せ本文領域（最大幅を持つ） |

## 4. デザイン基盤（トークン）

### タイポグラフィ
| 項目 | 値 |
|---|---|
| フォント | `"LINE Seed JP", system-ui, -apple-system, "Hiragino Sans", "Noto Sans JP", sans-serif`（主 = LINE Seed JP、LINE 提供の Web フォント。以降はフォールバック。読み込み方法は実装時） |
| 行送り（leading / line-height） | `1.7`（全体共通） |
| 字間（tracking / letter-spacing） | `0.02em`（全体共通） |
| ウェイト | 本文 400 / 強調・見出し 600 |

| サイズトークン | px | 用途 |
|---|---|---|
| text-xs | 13 | 補助・キャプション |
| text-sm | 15 | メタ情報（日時・状態ラベル） |
| text-base | 18 | 本文（既定） |
| text-lg | 20 | 強調本文・小見出し |
| text-xl | 24 | セクション見出し |
| text-2xl | 28 | サブページタイトル |
| text-3xl | 34 | ページタイトル |

### 余白（8px グリッド）
| トークン | px | 主な用途 |
|---|---|---|
| space-1 | 4 | アイコン-文字間など微調整 |
| space-2 | 8 | 要素内の小間隔 |
| space-3 | 12 | フォーム行内 |
| space-4 | 16 | 要素間の標準 |
| space-6 | 24 | 小ブロック間 |
| space-8 | 32 | セクション間 |
| space-12 | 48 | ページ上下の余白 |

### レイアウト
| 項目 | 値 |
|---|---|
| コンテナ最大幅（標準） | 960px（中央寄せ、左右 space-4 のパディング） |
| コンテナ最大幅（フォーム系） | 480px |
| ブレークポイント | `md = 768px`（モバイルファースト。未満=1カラム、以上=グリッド等を多カラム化） |

### 機能色（最小・無彩 + 機能アクセント）
ブランド配色（primary/accent 等）は持たない。以下の機能色のみ定義。

| トークン | 値 | 用途 |
|---|---|---|
| color-text | `#1a1a1a` | 本文・見出し |
| color-text-muted | `#6b7280` | 補助・メタ |
| color-border | `#d1d5db` | 罫線・カード境界・入力枠 |
| color-bg | `#ffffff` | 背景 |
| color-link | `#1d4ed8` | リンク（hover で下線） |
| color-error | `#dc2626` | エラー文・必須未入力・危険操作 |
| color-focus | `#1d4ed8` | フォーカスリング |
| color-disabled-bg | `#f3f4f6` | 無効要素の背景 |
| color-disabled-text | `#9ca3af` | 無効要素の文字 |

### フォーカス・操作性
| 項目 | 値 |
|---|---|
| フォーカスリング | `outline: 2px solid color-focus; outline-offset: 2px`（全インタラクティブ要素の `:focus-visible`） |
| タッチターゲット最小 | 操作要素 44×44px（モバイル）。インライン文字リンクは行高で確保し対象外 |

### 角丸
| トークン | 値 | 用途 |
|---|---|---|
| radius-sm | `4px` | 入力・ボタン・カード（大きい角丸は使わない） |

### コンポーネント寸法・状態
| 要素 | 寸法 | 状態 |
|---|---|---|
| ボタン | 高さ ~40px（縦 space-2〜3 / 横 space-4）、text-base、radius-sm | hover=下線または罫線濃く / focus=リング / disabled=disabled 色・操作不可 |
| 入力（text/textarea/select） | 高さ ~40px、padding space-2〜3、border 1px、radius-sm | focus=リング / error=border を color-error / disabled=disabled-bg |
| チェックボックス | 既定サイズ + label クリック可 | focus=リング |

### グリッド列数（レスポンシブ）
| 画面 | `<md`（モバイル） | `≥md` |
|---|---|---|
| 作品一覧/管理 | 1列（縦リスト） | 2〜3列グリッド |
| 公開ポートフォリオ | 1〜2列 | 3列グリッド |
| 検索結果 | 1列 | 作品=2列 / 作者=リスト |

## 5. 共通仕様（全画面に適用）

### 5.1 共通レイアウト / ナビゲーション
全ページ上部に共通ヘッダー。`<main>` はコンテナ内。

| 状態 | ヘッダー左 | ヘッダー右 |
|---|---|---|
| 未ログイン | サービス名（`/` へのリンク） | `検索`(/search) / `ログイン` / `登録` |
| ログイン済み | サービス名（`/` へのリンク） | `作品管理`(/artworks) / `検索`(/search) / `設定`(/settings) / `ログアウト` |

ヘッダーは全幅、`<main>` はコンテナ幅。`md` 未満では右側リンクを素直に折り返す（ハンバーガーは作らない）。

### 5.2 状態の共通パターン
| 状態 | 表示方針 |
|---|---|
| ローディング | 取得中は `読み込み中…` テキスト（装飾なし）。SSR 完結ページでは原則不要 |
| 空状態 | 一覧0件は説明文 + 主要アクションへの導線（例「作品がまだありません。新規作成」） |
| エラー | 操作失敗は `role="alert"` のテキストで操作近傍に表示。文言は具体的に |
| 404 | 該当なし（不正 slug/id）は「ページが見つかりません」+ 戻る導線。専用デザインは持たない |

### 5.3 遷移の原則
| 原則 | 内容 |
|---|---|
| 認証ガード | 保護ページに未ログインでアクセス → `/login` へ。成功後は元のページ or `/artworks` |
| 破壊的操作 | 削除は確認を挟む（ワイヤー段ではネイティブ confirm で可） |
| 送信後 | フォーム送信成功 → 関連一覧へ戻り結果反映（revalidate 済み） |

### 5.4 レスポンシブ方針
- モバイルファースト。`md` 未満は単一カラム・縦積み。`md` 以上で一覧/ポートフォリオをグリッド化（列数は §4 グリッド列数）。
- 画像は幅100%・アスペクト維持。一覧はサムネ幅、詳細は大サイズ（FR-15）。

### 5.5 アクセシビリティ（最低限）
- フォーム要素に `label`、エラーは `role="alert"`。
- 画像に `alt`（作品タイトル）。見出しは階層順（h1→h2）。
- フォーカスリングを全インタラクティブ要素に（§4）。
