# 02. プロトタイプ UI 仕様（ワイヤーフレーム品質）

## 1. 概要

01-prototype（機能・API・DB は実装済み）の上に、**全画面を「デザイン無し（ワイヤーフレーム品質）」で実装するための UI 仕様**を定める。視覚デザイン（ブランド配色・イラスト・凝った装飾）はこの段では作らず、**タイポグラフィ・余白・機能色などの最小基盤（トークン）だけを定義**して全画面に一貫適用する。機能は 01-prototype の要件を満たす範囲で、現状未実装の画面（横断検索・共通ナビ・ホーム）も含めて揃える。

### 成功条件
1. 本仕様の全画面が、要素・状態・遷移・レスポンシブ・文言の定義どおり実装される。
2. デザイン基盤トークン（フォント/サイズ/余白/機能色/レイアウト）が全画面に一貫適用される。
3. 装飾的ビジュアルデザインなしでも、各機能が迷わず操作・閲覧できる。

### 前提
| 前提 | 内容 |
|---|---|
| 機能基盤 | 認証・作品CRUD・画像・ポートフォリオ・検索の API/データは 01-prototype で実装済み。本仕様はその UI 層のみを扱う |
| 配色・ブランド | ブランド配色・装飾は作らない。ただし機能色（文字/罫線/エラー/リンク/フォーカス）の最小セットは定義して操作性を担保 |

## 2. スコープ

### In scope
- デザイン基盤トークン（タイポグラフィ・余白・レイアウト・機能色・フォーカス/操作性・角丸・コンポーネント寸法/状態・グリッド列数）
- 全画面共通のレイアウト/ナビゲーション、状態・遷移・レスポンシブの共通方針
- 各画面のワイヤーフレーム実装仕様（要素/レイアウト・状態・遷移・主要文言）

### Out of scope
- ブランド配色パレット・アイコンセット・影/エレベーション・凝ったモーション
- 新規ドメイン機能（課金・ソーシャル等）
- 画像編集・トリミング等の高度な操作 UI
- ダークモード
- 実装手段（Tailwind/CSS 設計）の確定 — トークンの「値」のみ定義し、適用手段は実装時

## 3. 用語

| 用語 | 意味 |
|---|---|
| トークン | フォントサイズ・余白・色などの再利用可能な定義値 |
| 空状態 | データが0件のときの表示 |
| コンテナ | 中央寄せ本文領域（最大幅を持つ） |
| Profile Slug | 公開プロフィール URL `/p/{slug}` のハンドル（`artist_profile.slug`）。UI では `@{slug}` 表記でリンク表示し、データ・URL には `@` を含めない |

## 4. デザイン基盤（トークン）

### タイポグラフィ
| 項目 | 値 |
|---|---|
| フォント | `"LINE Seed JP", system-ui, -apple-system, "Hiragino Sans", "Noto Sans JP", sans-serif`（主 = LINE Seed JP、LINE 提供の Web フォント。以降はフォールバック。読み込み方法は実装時） |
| 行送り（leading / line-height） | `1.7`（全体共通） |
| 字間（tracking / letter-spacing） | `0.02em`（全体共通） |
| ウェイト | 本文 400 / 強調・見出し 600 |

| サイズトークン | px | 用途 |
|---|---|---|
| text-xs | 13 | 補助・キャプション |
| text-sm | 15 | メタ情報（日時・状態ラベル） |
| text-base | 18 | 本文（既定） |
| text-lg | 20 | 強調本文・小見出し |
| text-xl | 24 | セクション見出し |
| text-2xl | 28 | サブページタイトル |
| text-3xl | 34 | ページタイトル |

### 余白（8px グリッド）
| トークン | px | 主な用途 |
|---|---|---|
| space-1 | 4 | アイコン-文字間など微調整 |
| space-2 | 8 | 要素内の小間隔 |
| space-3 | 12 | フォーム行内 |
| space-4 | 16 | 要素間の標準 |
| space-6 | 24 | 小ブロック間 |
| space-8 | 32 | セクション間 |
| space-12 | 48 | ページ上下の余白 |

### レイアウト
| 項目 | 値 |
|---|---|
| コンテナ最大幅（標準） | 960px（中央寄せ、左右 space-4 のパディング） |
| コンテナ最大幅（フォーム系） | 480px |
| ブレークポイント | `md = 768px`（モバイルファースト。未満=1カラム、以上=グリッド等を多カラム化） |

### 機能色（最小・無彩 + 機能アクセント）
ブランド配色（primary/accent 等）は持たない。以下の機能色のみ定義。

| トークン | 値 | 用途 |
|---|---|---|
| color-text | `#1a1a1a` | 本文・見出し |
| color-text-muted | `#6b7280` | 補助・メタ |
| color-border | `#d1d5db` | 罫線・カード境界・入力枠 |
| color-bg | `#ffffff` | 背景 |
| color-link | `#1d4ed8` | リンク（hover で下線） |
| color-error | `#dc2626` | エラー文・必須未入力・危険操作 |
| color-focus | `#1d4ed8` | フォーカスリング |
| color-disabled-bg | `#f3f4f6` | 無効要素の背景 |
| color-disabled-text | `#9ca3af` | 無効要素の文字 |

### フォーカス・操作性
| 項目 | 値 |
|---|---|
| フォーカスリング | `outline: 2px solid color-focus; outline-offset: 2px`（全インタラクティブ要素の `:focus-visible`） |
| タッチターゲット最小 | 操作要素 44×44px（モバイル）。インライン文字リンクは行高で確保し対象外 |

### 角丸
| トークン | 値 | 用途 |
|---|---|---|
| radius-sm | `4px` | 入力・ボタン・カード（大きい角丸は使わない） |

### コンポーネント寸法・状態
| 要素 | 寸法 | 状態 |
|---|---|---|
| ボタン | 高さ ~40px（縦 space-2〜3 / 横 space-4）、text-base、radius-sm | hover=下線または罫線濃く / focus=リング / disabled=disabled 色・操作不可 |
| 入力（text/textarea/select） | 高さ ~40px、padding space-2〜3、border 1px、radius-sm | focus=リング / error=border を color-error / disabled=disabled-bg |
| チェックボックス | 既定サイズ + label クリック可 | focus=リング |

### グリッド列数（レスポンシブ）
| 画面 | `<md`（モバイル） | `≥md` |
|---|---|---|
| 作品一覧/管理 | 1列（縦リスト） | 2〜3列グリッド |
| 公開ポートフォリオ | 1〜2列 | 3列グリッド |
| 検索結果 | 1列 | 作品=2列 / 作者=リスト |

## 5. 共通仕様（全画面に適用）

### 5.1 共通レイアウト / ナビゲーション
全ページ上部に共通ヘッダー。`<main>` はコンテナ内。

| 状態 | ヘッダー左 | ヘッダー右 |
|---|---|---|
| 未ログイン | サービス名（`/` へのリンク） | `検索`(/search) / `ログイン` / `登録` |
| ログイン済み | サービス名（`/` へのリンク） | `作品管理`(/artworks) / `検索`(/search) / `設定`(/settings) / `ログアウト` |

ヘッダーは全幅、`<main>` はコンテナ幅。`md` 未満では右側リンクを素直に折り返す（ハンバーガーは作らない）。

### 5.2 状態の共通パターン
| 状態 | 表示方針 |
|---|---|
| ローディング | 取得中は `読み込み中…` テキスト（装飾なし）。SSR 完結ページでは原則不要 |
| 空状態 | 一覧0件は説明文 + 主要アクションへの導線（例「作品がまだありません。新規作成」） |
| エラー | 操作失敗は `role="alert"` のテキストで操作近傍に表示。文言は具体的に |
| 404 | 該当なし（不正 slug/id）は「ページが見つかりません」+ 戻る導線。専用デザインは持たない |

### 5.3 遷移の原則
| 原則 | 内容 |
|---|---|
| 認証ガード | 保護ページに未ログインでアクセス → `/login` へ。成功後は元のページ or `/artworks` |
| 破壊的操作 | 削除は確認を挟む（ワイヤー段ではネイティブ confirm で可） |
| 送信後 | フォーム送信成功 → 関連一覧へ戻り結果反映（revalidate 済み） |

### 5.4 レスポンシブ方針
- モバイルファースト。`md` 未満は単一カラム・縦積み。`md` 以上で一覧/ポートフォリオをグリッド化（列数は §4 グリッド列数）。
- 画像は幅100%・アスペクト維持。一覧はサムネ幅、詳細は大サイズ（FR-15）。

### 5.5 アクセシビリティ（最低限）
- フォーム要素に `label`、エラーは `role="alert"`。
- 画像に `alt`（作品タイトル）。見出しは階層順（h1→h2）。
- フォーカスリングを全インタラクティブ要素に（§4）。

## 6. 画面仕様

各画面: 目的 / 主な要素 / 状態 / 遷移 / 主要文言。共通事項（ヘッダー・状態パターン）は §5 を参照し再掲しない。

### 6.1 ホーム `/`
| 項目 | 内容 |
|---|---|
| 目的 | サービスの入口。未ログインは簡易案内、ログイン済みは管理へ誘導 |
| 挙動 | ログイン済みは `/artworks` へリダイレクト。未ログインはランディング表示 |
| 要素（未ログイン） | サービス名（h1）/ 一言説明 / `ログイン`・`登録` ボタン |
| 文言 | 見出し「アートワーク」 / 説明「作品を登録して公開ポートフォリオを作る」 |

### 6.2 ログイン `/login`
| 項目 | 内容 |
|---|---|
| 目的 | 既存ユーザーのログイン |
| 要素 | h1「ログイン」/ メール入力 / パスワード入力 / `ログイン`ボタン / `サインアップへ`リンク |
| 状態 | 入力エラー（形式/必須）はフィールド近傍 / 認証失敗は `role="alert"`「メールまたはパスワードが違います」 |
| 遷移 | 成功 → `/artworks` / 「サインアップへ」→ `/signup` |

### 6.3 サインアップ `/signup`
| 項目 | 内容 |
|---|---|
| 目的 | 新規登録（直後に artist_profile 初期化＝FR-03） |
| 要素 | h1「登録」/ 表示名 / メール / パスワード(8文字以上) / `登録`ボタン / `ログインへ`リンク |
| 状態 | バリデーション（メール形式・パスワード長・必須）/ 重複メールは `role="alert"`「このメールは登録済みです」 |
| 遷移 | 成功 → `/artworks` / 「ログインへ」→ `/login` |

### 6.4 ログアウト `/logout`
| 項目 | 内容 |
|---|---|
| 目的 | セッション終了 |
| 要素 | 確認テキスト + `ログアウト`ボタン（または即時処理） |
| 遷移 | 実行後 `/login` |

### 作品の状態モデル（status）

作品のライフサイクルを単一の `status` で表す。公開は `status` で決まり、ポートフォリオ掲載（§6.12）は別概念。

| 項目 | 内容 |
|---|---|
| フィールド | `artwork.status` enum = `draft`（下書き・非公開）/ `published`（公開）/ `archived`（取り下げ・非公開、管理一覧には残す）。新規作成時は `draft` |
| 公開 | **公開（検索・公開ページに出る）＝ `status='published'`**。`draft`/`archived` は非公開 |
| 作成 | `/artworks/new` を開いた時点で作品（status=draft・タイトル空可）を作成し `/artworks/edit/[id]` へ遷移。新規作成は常に新しい下書きを作る |
| 自動保存 | 編集画面のタイトル・説明・状態(status)は変更時に PATCH で随時保存。画像は選択時に保存。**保存ボタンは持たない** |
| 公開/非公開 | status を `published` にすると公開（このときタイトル必須）。`draft`/`archived` に戻すと非公開 |
| ポートフォリオ | 公開（published）作品の中から `/portfolio/edit` で選んで並べる（§6.12） |
| 再開/削除 | 一覧の作品を選ぶと続きから編集。空の下書きは自動削除せず手動削除（削除で R2 も掃除＝FR-07） |
| 販売状態 | 販売中/取引中/販売済 等の販売状態カラムは本仕様では持たない（項目未確定・将来追加） |

### 6.5 作品一覧 / 管理 `/artworks`（要ログイン）
| 項目 | 内容 |
|---|---|
| 目的 | 自分の全作品の一覧と管理導線（draft/published/archived 含む） |
| 要素 | h1「作品管理」/ `新規作成`リンク / 作品リスト（各項目: サムネ・タイトル・状態(下書き/公開/アーカイブ)・`編集`・`削除`）。**非公開（draft/archived）はカード右上に状態バッジ（`draft`/`archived`）を表示** |
| レイアウト | §4 グリッド列数（`<md`=縦リスト / `≥md`=2〜3列） |
| 状態 | 空: 「作品がまだありません。新規作成」 / 削除中: 当該項目を操作不可に |
| 遷移 | 新規作成→`/artworks/new`（下書き作成し編集へ） / 編集→`/artworks/edit/[id]` / 削除→確認後その場で除去（R2画像も削除＝FR-07） |

### 6.6 作品作成 `/artworks/new`（要ログイン）
| 項目 | 内容 |
|---|---|
| 目的 | 新規下書きの作成と編集画面への遷移 |
| 挙動 | アクセス時に作品（status=draft・タイトル空）を1件作成し、`/artworks/edit/[id]` へリダイレクト。フォームは持たない（編集画面に集約）。新規作成は常に新しい下書きを作る |

### 6.7 作品編集 `/artworks/edit/[id]`（要ログイン）
| 項目 | 内容 |
|---|---|
| 目的 | 作品の編集（所有者のみ）。状態(status)の変更もここで行う |
| 要素 | h1「作品を編集」/ タイトル / 説明 / 状態(下書き/公開/アーカイブ select) / 画像アップロード(複数可・並び替え) / `一覧へ戻る`。**掲載チェックボックス・保存ボタンは無し（自動保存）** |
| 自動保存 | タイトル・説明・状態(status)は変更（blur/change）時に PATCH で随時保存。画像は選択時に保存（NFR-02） |
| 状態 | 下書きはタイトル空可。status を `published` にする時はタイトル必須（空なら不可・フィールドエラー）。他人の作品/存在しない → 404「作品が見つかりません」 |
| 遷移 | `一覧へ戻る` → `/artworks`（変更は自動保存済み） |

### 6.8 設定 `/settings`（要ログイン）
| 項目 | 内容 |
|---|---|
| 目的 | プロフィールと Profile Slug（公開URLハンドル）の編集 |
| 要素 | h1「設定」/ 表示名 / **Profile Slug**（入力は `@` プレフィックス表示・保持値に `@` は含めない・重複/形式チェック）/ 自己紹介(bio) / `保存`ボタン / `ポートフォリオ編集`(/portfolio/edit) と 自分のページ（`@{slug}` 表記で `/p/{slug}` へのリンク） |
| 状態 | slug 重複は `role="alert"`「この slug は使用されています」/ 形式不正はフィールドエラー |
| 遷移 | 保存 → 同画面で結果表示（slug 変更時は新URL を案内）/ `ポートフォリオ編集` → `/portfolio/edit` |

公開制御は作品の `status`（published＝公開）で行う。ポートフォリオ掲載は `/portfolio/edit`（§6.12）で選ぶ。

### 6.9 検索結果 `/search`（公開）
| 項目 | 内容 |
|---|---|
| 目的 | 作品・作者の横断検索（FR-17、pg_trgm 部分一致） |
| 検索対象 | 公開作品（`status='published'`、title/description の部分一致）と 公開プロフィール（display_name）。draft/archived は対象外 |
| 要素 | 検索ボックス（クエリ `q`）/ `作品` 結果（サムネ+タイトル、`/p/{slug}/{id}` へ）/ `作者` 結果（表示名+slug、`/p/{slug}` へ） |
| レイアウト | §4 グリッド列数（作品=`≥md` で2列、作者=リスト） |
| 状態 | 空クエリ: 結果非表示・プロンプト「キーワードを入力」/ 0件: 「該当がありません」 |
| 遷移 | ヘッダーの `検索` リンク（全状態で表示）or 直接アクセス。結果クリック → 各公開ページ |

### 6.10 公開ポートフォリオ `/p/[slug]`（公開・未認証）
| 項目 | 内容 |
|---|---|
| 目的 | 作者のポートフォリオ（選んだ公開作品、SSR + キャッシュ、NFR-06） |
| 要素 | h1「作者表示名」/ 自己紹介(bio, 任意) / ポートフォリオ作品グリッド（サムネ+タイトル、`/p/{slug}/{id}` へ） |
| 表示条件 | `status='published'` かつ **ポートフォリオに選択された作品**・選択順（position）。§6.12 で編集（FR-12,13） |
| レイアウト | §4 グリッド列数（`<md`=1〜2列 / `≥md`=3列） |
| 状態 | slug 該当なし → 404「ポートフォリオが見つかりません」/ 公開作品0件 → 「公開作品はまだありません」 |
| SEO | title=表示名、OGP 画像=先頭作品の先頭画像（FR-16） |

### 6.11 公開作品詳細 `/p/[slug]/[artworkId]`（公開・未認証）
| 項目 | 内容 |
|---|---|
| 目的 | 公開作品1点の詳細閲覧 |
| 表示条件 | `status='published'`（公開）なら閲覧可。ポートフォリオ未選択でも published なら直リンク・検索から到達できる |
| 要素 | h1「作品タイトル」/ 説明 / 画像（詳細用・大サイズ、複数は縦並び）/ `{作者}のポートフォリオへ戻る`リンク |
| 状態 | 非公開（draft/archived）/存在しない → 404「作品が見つかりません」 |
| SEO | title=「作品名 - 作者名」、OGP=先頭画像 |

### 6.12 ポートフォリオ編集 `/portfolio/edit`（要ログイン・新規）
| 項目 | 内容 |
|---|---|
| 目的 | 公開ポートフォリオ（`/p/[slug]`）に載せる作品の選択と並び替え（1人1ポートフォリオ） |
| 要素 | h1「ポートフォリオ編集」/ 自分の**公開（published）作品**のリスト（各: サムネ・タイトル・`掲載`チェック・並び替え `↑`/`↓`）/ `保存`ボタン / 公開ページ(`/p/{slug}`) と 設定 へのリンク |
| 振る舞い | 掲載チェックで選択、`↑`/`↓` で選択内の表示順を変更。`保存`で選択集合＋順序を確定（公開ページに反映） |
| 状態 | 公開作品が0件: 「公開作品がありません。作品を公開すると掲載できます」（`/artworks` への導線） |
| 遷移 | 保存 → 同画面で結果反映 / `公開ページを見る` → `/p/{slug}` |
