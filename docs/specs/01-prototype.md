# プロトタイプ 仕様書

---

## 1. 概要

アーティストが作品を登録・管理し、公開ポートフォリオとして見せられる最小の動くプロトタイプを作る。
課金は含めず、プロダクトのコア体験（登録 → 公開 → 共有）を検証する。

### 成功条件

1. アーティストがサインアップ／ログインできる
2. 作品を画像付きで登録・編集・削除できる
3. 公開ポートフォリオページが生成され、公開設定ができる

---

## 2. スコープ

- メール+パスワード認証（Better Auth）
- 作品の CRUD（画像 1〜N 枚、メタデータ）
- 画像アップロード（R2）と配信（Cloudflare Images 変換）
- 公開ポートフォリオページ（slug ベースの公開 URL）

---

## 3. 用語

| 用語 | 意味 |
|---|---|
| アーティスト（artist） | 本サービスの登録ユーザー。作品を持つ主体 |
| 作品（artwork） | アーティストが登録する1つの作品。1枚以上の画像とメタデータを持つ |
| 作品画像（artwork image） | 作品に紐づく画像。R2 に保存し Cloudflare Images で配信 |
| ポートフォリオ（portfolio） | アーティスト単位の公開ページ。公開作品の集合 |

---

## 4. ユーザーと役割

- **アーティスト（認証ユーザー）**: 自分の作品とポートフォリオを管理
- **閲覧者（匿名）**: 公開ポートフォリオを閲覧するだけ。認証なし

## 6. 機能要件

| ID | 分類 | 機能要件 | 備考 |
|---|---|---|---|
| FR-01 | 認証 | メール+パスワードでのサインアップ / ログイン / ログアウト（Better Auth） | |
| FR-02 | 認証 | セッションで API のアーティストを識別 | |
| FR-03 | 認証 | サインアップ直後に `artist_profile` を初期化 | slug は仮値 → 設定画面で変更 |
| FR-04 | 認証 | メール確認（verification）を必須にするか | `要確認`。プロトタイプは任意で開始を推奨 |
| FR-05 | 作品管理 | アーティストは自分の作品を一覧・作成・編集・削除できる | |
| FR-06 | 作品管理 | 作成/編集で画像を1枚以上アップロードできる（複数可、並び替え可） | アップロード方式は下記 |
| FR-07 | 作品管理 | 削除時は紐づく画像（R2 オブジェクト含む）も削除する | |
| FR-08 | 作品管理 | `status` で 下書き / 公開 / アーカイブ を切り替えられる | `draft` / `published` / `archived` |
| FR-09 | 作品管理 | 作品の表示順（`sort_order`）を変更できる | |
| FR-10 | 作品管理 | 所有者一致をサーバー側（API）で必ず検証する | 認可（7.3） |
| FR-11 | ポートフォリオ | 公開 URL `/p/{slug}` でポートフォリオを表示 | `artist_profile.slug` |
| FR-12 | ポートフォリオ | `status = published` かつ ポートフォリオに掲載選択された作品のみ表示 | 掲載選択は `portfolio_item` |
| FR-13 | ポートフォリオ | 作品をポートフォリオの選択順（`portfolio_item.position`）に並べる | |
| FR-14 | ポートフォリオ | 作品詳細の公開ページ `/p/{slug}/{artwork_id}` | プロトタイプは id で開始 |
| FR-15 | ポートフォリオ | 画像は Cloudflare Images 変換で適切なサイズ配信 | 一覧はサムネ、詳細は大。高解像度は不要（NFR-04） |
| FR-16 | ポートフォリオ | 最低限の SEO / OGP（meta / OGP 画像は先頭画像） | |
| FR-17 | 検索 | ユーザーは作品・作者・ポートフォリオを横断検索できる | 実現方式は NFR-05 |

画像アップロード方式（FR-06、詳細は 7.1）:

1. クライアント → API に署名付き URL を要求
2. クライアント → R2 に直接 PUT（API を経由しない）
3. 完了後、API に `r2_key` を通知して `artwork_image` を作成

## 7. 非機能・技術的論点

| ID | 分類 | 要件・方針 | 備考 |
|---|---|---|---|
| NFR-01 | 画像処理・配信 | 原本は R2 のみで管理 | 低メンテ方針 |
| NFR-02 | 画像処理・配信 | アップロードはブラウザから R2 へ直接（署名付き URL、API 非経由） | Workers のリクエストサイズ制約を回避 |
| NFR-03 | 画像処理・配信 | 取得はブラウザから直接、R2 → Image Resizing の経路で配信時にオンザフライ変換 | 同期処理に依存しない。画像ロジック `apps/api/src/lib/image/`、R2 抽象 `apps/api/src/lib/storage.ts`（apps/api 規約） |
| NFR-04 | 画像処理・配信 | 画像は**一般的な閲覧解像度**で扱い、印刷用などの高解像度は保持・配信しない。アップロード時に長辺の上限までリサイズする | 一般用途で十分。ストレージ / 転送コスト削減。上限値は `要確認`（例: 長辺 2048px） |
| NFR-05 | 検索 | 検索（FR-17）は PostgreSQL の pg_trgm + GIN で実現し、日本語を含む部分一致をカバーする | migration に手書き（DB 規約）。高精度な日本語全文検索が要れば外部エンジン等を別途検討 |
| NFR-06 | パフォーマンス | 公開ポートフォリオは SSR + キャッシュ前提で高速表示。作品更新時に `revalidateTag` で無効化 | 閲覧者は未認証・読み取り中心 |
| NFR-07 | パフォーマンス | 画像は用途別サイズで配信（一覧サムネ / 詳細大）+ CDN キャッシュ。過大な原本を直接配らない | FR-15 / NFR-03 / NFR-04 |
| NFR-08 | パフォーマンス | 作品一覧・ポートフォリオは作品数が増えても破綻しないようページング / 遅延読み込み | 目標値は `要確認`（例: 初期表示 LCP < 2.5s） |
| NFR-09 | Workers 制約 | Node.js built-in 非対応。DB は `@artwork/database` の `createDb()`（neon-http） | |
| NFR-10 | Workers 制約 | web 側キャッシュは `unstable_cache` + `revalidateTag`（`use cache` は使わない） | apps/web 規約 |
| NFR-11 | API 構成 | `apps/api` を Hono RPC で公開し、型を `@artwork/shared` 経由で web と共有 | 主要ルートは下記 |

主要ルート（草案、NFR-11）:

- `POST /artworks` / `GET /artworks` / `GET /artworks/:id` / `PATCH /artworks/:id` / `DELETE /artworks/:id`
- `POST /artworks/:id/images`（メタ作成） / `DELETE /images/:id` / 並び替え
- `POST /uploads/sign`（R2 署名付き URL 発行）
- `GET /portfolio/:slug`（公開・未認証）

## 8. セキュリティ要件

| ID | 分類 | 要件・方針 | 備考 |
|---|---|---|---|
| SEC-01 | 認可 | 所有者一致をサーバー側（API）で必ず検証する | 要件の正は FR-10。本章は横断的な集約 |
| SEC-02 | 認可（実装手段） | DB 層でも認可を強制する RLS を入れるかは段階導入で判断 | `要確認`。プロトタイプは SEC-01 のアプリ層認可で開始（未知を抱えない方針）。RLS は多層防御の追加であり SEC-01 を代替しない |
| SEC-03 | 認証 | Better Auth セッションでアーティストを識別。セッション Cookie は同一オリジンで `SameSite=Lax` | |
| SEC-04 | 機密情報 | 機密は api Worker に集約（`DATABASE_URL` / `BETTER_AUTH_SECRET` / R2 の S3 キー等）。web 側に機密を置かない | |
| SEC-05 | 通信 | 全通信を HTTPS / TLS（Cloudflare 終端） | |
| SEC-06 | アップロード | R2 への直接アップロードは短命・スコープ限定の署名付き URL を発行する | NFR-02 と対 |

## 9. 画面構成（web、草案）

認証エリア（要ログイン）:

- `/artwroks/list` — 作品一覧 / 管理
- `/artwroks/new` `/artworks/edit/:id` — 作成・編集
- `/settings` — プロフィール / slug / 公開設定

公開エリア（未認証）:

- `/p/:slug` — ポートフォリオ
- `/artworks/:id` — 作品詳細
- `/login` `/signup`
